# Plugin Catalog

Plugin catalog work has started. This slice validates, imports, stores, lists, enables, and disables approved plugin manifests, but it does not execute plugins, publish plugin commands, scrape docs, or route plugin prefix commands yet.

## Current Behavior

- plugin manifests can be decoded from `gigi-plugin.json`
- manifests must declare exact Discord application ID or bot user ID
- manifests can come from the known catalog, an explicitly provided HTTPS manifest URL, or a Discord attachment upload
- approved manifests can be stored in PostgreSQL
- `/plugins list` shows approved manifests
- `/plugins import-manifest` imports and approves a URL-sourced manifest
- `/plugins import-file` imports and approves an uploaded JSON manifest
- `/plugins enable` enables an approved plugin version for the current guild
- `/plugins disable` disables a plugin for the current guild
- `/plugins enabled` shows enabled guild plugins
- enabled guild plugin installs can be loaded from PostgreSQL

v0 does not guess plugin identity from fuzzy names, bot display names, domains, or docs pages. Fuzzy suggestions may come later, but exact admin approval remains required.

Future manifest contributions should follow the same runtime format and safety rules in the [contributor guide](/contributing).

## Manifest Requirements

Each manifest must include:

- plugin id, name, version, source, and source kind
- exact Discord application ID or bot user ID
- capabilities
- triggers
- supported surfaces
- required Gigi capabilities
- audit events
- attribution resources

Manifest URL imports must use HTTPS and cannot include user info, query strings, or fragments. Attachment imports must use a `.json` file and are stored as `uploaded_file` source without storing the Discord CDN URL. The current code validates supplied manifest JSON; it does not browse or scrape plugin documentation.

```json
{
  "id": "example-tool",
  "name": "Example Tool",
  "version": "1.0.0",
  "source": "builtin",
  "source_kind": "known",
  "discord_application_id": "1511678703963209813",
  "discord_bot_user_id": "1511678703963209814",
  "capabilities": [
    {
      "name": "example.run",
      "description": "Run the plugin's declared action."
    }
  ],
  "triggers": [
    {
      "kind": "prefix",
      "value": "!example"
    }
  ],
  "surfaces": ["guild_text"],
  "permissions": ["example.run"],
  "audit_events": ["plugin.example.run"],
  "attribution": [
    {
      "name": "Example Provider",
      "use": "Provide data for the plugin action.",
      "source": "https://example.com"
    }
  ]
}
```

## Not Live Yet

- admin-assisted custom manifest creation
- community registry support
- plugin command publishing
- prefix trigger routing
- LLM tool routing
- plugin execution
- docs scraping or browser-based discovery

Next slice should expose a permission-gated admin flow to list known plugins and enable an approved manifest for a guild.
