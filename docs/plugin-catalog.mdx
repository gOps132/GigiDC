# Plugin Catalog

Plugin catalog work has started. This slice validates and stores approved plugin manifests, but it does not execute plugins, publish plugin commands, scrape docs, or route plugin prefix commands yet.

## Current Behavior

- plugin manifests can be decoded from `gigi-plugin.json`
- manifests must declare exact Discord application ID or bot user ID
- manifests can come from the known catalog or from an explicitly provided manifest URL
- approved manifests can be stored in PostgreSQL
- enabled guild plugin installs can be loaded from PostgreSQL

V1 does not guess plugin identity from fuzzy names, bot display names, domains, or docs pages. Fuzzy suggestions may come later, but exact admin approval remains required.

## Manifest Requirements

Each manifest must include:

- plugin id, name, version, source, and source kind
- exact Discord application ID or bot user ID
- capabilities
- triggers
- supported surfaces
- required Gigi capabilities
- audit events
- attribution resources

Manifest URL imports must use HTTPS. The current code validates supplied manifest JSON; it does not browse or scrape plugin documentation.

```json
{
  "id": "example-tool",
  "name": "Example Tool",
  "version": "1.0.0",
  "source": "builtin",
  "source_kind": "known",
  "discord_application_id": "1511678703963209813",
  "discord_bot_user_id": "1511678703963209814",
  "capabilities": [
    {
      "name": "example.run",
      "description": "Run the plugin's declared action."
    }
  ],
  "triggers": [
    {
      "kind": "prefix",
      "value": "!example"
    }
  ],
  "surfaces": ["guild_text"],
  "permissions": ["example.run"],
  "audit_events": ["plugin.example.run"],
  "attribution": [
    {
      "name": "Example Provider",
      "use": "Provide data for the plugin action.",
      "source": "https://example.com"
    }
  ]
}
```

## Not Live Yet

- Discord `/plugins` admin commands
- guild enable/configure command flow
- plugin command publishing
- prefix trigger routing
- LLM tool routing
- plugin execution
- docs scraping or browser-based discovery

Next slice should expose a permission-gated admin flow to list known plugins and enable an approved manifest for a guild.
